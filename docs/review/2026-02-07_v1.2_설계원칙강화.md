# 교차검증: v1.1 → v1.2 설계 원칙 강화

- 날짜: 2026-02-07
- 검증 대상: README.md, CLAUDE.md (v1.1)
- 사용 도구: ChatGPT (2차 리뷰)

## 검증 내용
v1.1 교차검증 반영 후, 3가지 추가 설계 원칙을 지시받음.

## 반영 항목

| # | 지시 | 반영 위치 | 상세 |
|---|------|----------|------|
| 1 | `is_before_base`, `will_extinguish` 계산 책임을 RegistryAnalyzer로 이전 | CLAUDE.md (절대원칙 #5, 데이터 모델, 모듈 상세, DB 스키마), README (구조, 체크리스트, 계산 책임 다이어그램) | RuleEngine은 파생 필드 소비만. 계산 로직은 parser/registry_analyzer.py에 집중 |
| 2 | 애매한 Hard Stop → Yellow Zone 강등 | CLAUDE.md (절대원칙 #6, Hard Stop 섹션, 금기사항 #8), README (Hard Stop 섹션) | 교차검증 충돌 시 확실한 것만 Hard Stop 유지 |
| 3 | Public에서 '예측/적정가/입찰가' 금지 | CLAUDE.md (절대원칙 #7, 금기사항 #9, banned_phrases.json, Public 금지 용어 테이블), README (API 설명, Public 금지 용어 테이블) | 대체 표현 정의 완료 |

## 설계 결정 근거
- **#1**: 단일 책임 원칙(SRP). 등기부 분석 로직이 RuleEngine에 섞이면 테스트/유지보수가 어려워짐
- **#2**: 보수적 원칙. 확실하지 않은 Hard Stop은 좋은 물건을 놓치게 하므로, Yellow Zone에서 경고 후 사용자가 판단
- **#3**: 법적 안전. Public 서비스에서 투자자문 의혹을 원천 차단

## 결론
3건 모두 즉시 반영. 이후 추가 교차검증에서도 Hard Stop 강등 규칙을 적용하여, 불확실한 조건이 Hard Stop에 남지 않도록 관리한다.
